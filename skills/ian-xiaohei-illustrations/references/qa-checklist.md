# QA Checklist

## Must-pass criteria

- It is 16:9 horizontal.
- Background is clean white.
- Xiaohei is present.
- Xiaohei performs the core action and is not merely decorative.
- No old example composition was copied; a new metaphor was invented for the current article.
- The image is absurd, creative, and interesting.
- Clean and uncluttered; main subject does not exceed ~60% of the canvas.
- One image communicates only one core structure.
- Annotations are few, short, and readable.
- Orange is used only for the main path or arrows.
- Red is used only for highlights, problems, warnings, or results.
- Blue is used only for secondary notes, feedback, or system state.

## Failure signals

If any of these appear, regenerate or edit locally:

- Top-left corner has a type label such as "Common Pitfalls / Workflow / System Architecture / Roadmap."
- Xiaohei looks like a mascot, emoji character, or cute cartoon.
- The image looks like a PPT slide, course material, or formal flowchart.
- Too many elements, arrows, or nodes.
- Text becomes a block of explanation.
- Background has paper texture, shadows, gradients, off-white tone, or noise.
- Real UI screenshots or high-tech interface elements appear.
- Annotation text is badly garbled or unreadable.
- The image is too rigid with no absurd metaphor.
- The composition is too similar to an existing example in `assets/examples/`.

## Iteration methods

- Too ordinary: make Xiaohei the action subject; add one strange but plausible metaphor.
- Too complex: remove nodes; keep only one action and 3-5 short labels.
- Too cute: emphasize deadpan, blank serious expression, not cute, not mascot.
- Too PPT: remove the title, borders, tidy grid, and excess arrows; switch to a hand-drawn scene.
- Too similar to an old example: keep the core meaning; swap the main object and Xiaohei's action.
- Text errors: prefer local editing first; if errors are severe, regenerate with fewer labels.

## Delivery quality standard

A high-quality image should make the viewer think "this is a bit strange" first, then understand the structure within one second.

If the first impression is "tutorial page" rather than "absurd product sketch on blank paper," it does not pass.
