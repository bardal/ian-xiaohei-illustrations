# Composition Patterns and Originality Rules

## Core structure types

Choose one structure per image; do not mix too many.

### Workflow

Best for: input → process → output, content production, AI workflows, automation pipelines.

Layout: input on the left, Xiaohei or a strange machine in the middle processing, output on the right, orange arrows for main flow direction.

### System slice

Best for: information sources, filters, databases, renderers, partial views of an agent system.

Layout: only 3-5 core modules; Xiaohei participates in one key action.

### Before/after contrast

Best for: chaos vs. order, manual vs. automated, scattered vs. consolidated, anxious vs. stable.

Layout: left side chaotic, right side stable, orange arrow in the middle. Characters can be more exaggerated.

### Role state

Best for: user pain points, creator states, tool overload, information anxiety, stuck-to-running transitions.

Layout: 2-4 small states, each with a short label.

### Conceptual metaphor

Best for: content factory, information warehouse, mental black box, workflow machine, auto-generated daily report.

Layout: one large strange object or machine, minimal input, one output. Should be memorable.

### Method layers

Best for: methodology frameworks, system hierarchies, capability stacks, content system layers.

Layout: stacked boxes, not a formal pyramid; Xiaohei laying bricks or building beside them.

### Map / route

Best for: idea-to-launch journey, user path, content handoff path, learning roadmap.

Layout: one winding path, few nodes, Xiaohei pulling a line or walking.

### Mini comic strip

Best for: failure-to-success arc, real process depiction, complaint, before/after product usage.

Layout: 2-4 small panels, each expressing only one action.

## Original metaphor method

Invent a new metaphor from the current article every time. Never copy from old images.

### Three steps

1. Translate the abstract concept into a physical action: stuck, leaking, getting heavier, sorting, settling, fermenting, opening a door, folding, unpacking, flowing back.
2. Translate the system structure into a low-tech object: broken machine, cardboard box, drawer, water pipe, mailbox, strange dial, scale, well, ladder, weird workstation.
3. Give Xiaohei the action: not standing beside it, but stuck inside the machine, pulling the wrong wire, guarding the door, hauling, patching, weighing, holding the ladder, recording, pushing something into a strange device.

### Available object pool

- Cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, water pipe, ball of wire, sluice gate, turntable, black box, hole punch, noodle press, clothesline, odd workstation.
- Use only 1-2 per image; do not crowd the scene.

### Xiaohei action pool

- Pull, carry, stuff, scoop, press, weigh, sew, cut, twist, guard, push, receive, disassemble, label, recycle.
- Actions must serve the core meaning; do not use strange actions just for strangeness.

## No-copy rule

Do not default to opening or copying from `assets/examples/`. Those images exist only for style calibration — line density, white space, color restraint, and the quality of Xiaohei's participation.

Unless the user explicitly says "copy this / replicate this composition / use this example," never directly reuse these known compositions:

- Conveyor belt with two breakpoints
- Xiaohei pulling a decision lever inside a content machine
- Xiaohei becoming a funnel sorting traffic / trust / conversion
- Xiaohei cutting a material fish
- Xiaohei leading a handoff path
- Xiaohei pulling three-layer information sources
- Three Xiaoheis each holding a megaphone / building a bridge / opening a door
- Xiaohei stamping a handoff-script toolbox
- Xiaohei holding a sign near a common-pitfalls path

Even similar themes require a fresh metaphor. For example, "handoff path" does not have to be a road — it could be Xiaohei tying the tail of a piece of content to a door handle; "one fish many uses" does not have to be a fish — it could be Xiaohei pressing a paper ball into several different shapes.
